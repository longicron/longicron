vulnhub靶机实践之DC-9

vulnhub
靶机writeup
发布日期:   2020-12-08

一.环境搭建

下载链接：https://www.vulnhub.com/entry/dc-9,412/

解压后用Virtual Box打开，导入虚拟机

网卡配置看个人习惯，我喜欢桥接

二.信息搜集

1.netdiscover扫描靶机IP

netdiscover -r 192.168.8.0/24

可以根据虚拟机的mac地址来寻找对应的IP地址为192.168.8.13

2.nmap扫描端口

nmap -p- -Pn -sV -n 192.168.9.69

3.扫描web目录

dirb http://192.168.8.13

扫描后没有什么发现

三.漏洞发现

1.那就直接登录80端口上去看看吧



这边看了有搜索功能及登录功能，第一步想到sqlmap扫描是否有sql注入漏洞
搜索某个名字使用burp截包，保存为文件，使用sqlmap扫描



爆库，发现admin用户的密码，看上去像32为的md5密文，网上cmd5或者somd5解密一波，密码为transorbital1

另外爆了StaffDetails表的数据，发现有所有人员的用户名及密码，后面可能就是ssh登录的用户名及密码

2.使用admin及密码登录web页面

发现没有什么特别的地方，不过File does not exist这样的提示很有暗示性。

那我们就在url后面加上?file=/etc/passwd，发现读取不了，那就进行路径遍历

3.打开22端口

前面我们nmap扫描的结果显示ssh端口22是filter，流量被过滤了。联想到可能是/etc/knockd.conf对ssh进行了防护
我们使用文件包含查看下/etc/knockd.conf的内容

4.端口敲门

knockd字面意思是敲，只是这里敲的是端口，而且需要按照顺序‘敲’端口。如果敲击规则匹配，则可以让防火墙实时更改策略。从而达到开关防火墙的目的。

那我们按照/etc/knockd.conf文件中的自定义端口，根据Port-knocking的规则依次访问这三个端口就可以打开SSH服务了

我们再次扫描下22端口，发现已经打开

四.漏洞利用

1.用户ssh登录

根据sqlmap爆出来的用户名及密码，使用hydra进行用户名及密码爆破

hydra -L username.txt -P password.txt ssh://192.168.8.13


挨个进行登录，发现janitor/Ilovepeepee登录后在家目录下发现一些密码

加入到密码文件中继续ssh爆破，得到一个新的用户名及密码可登录

2.使用新的用户名及密码登录继续探索

进去后，一顿查找没有什么可疑文件
分别进行一下尝试，均无果
a.内核提权
b.suid提权
c.计划项提权

3.sudo提权

最后试了下sudo提权，先sudo -l查看配置项，发现问题

尝试运行了是一个python文件，找到该文件，发现脚本的功能是从一个文件读内容写到另一个文件中

那我们有了如下思路：可以构造一个root权限用户，将该用户信息写入文件/tmp/passwd内，将该文件作为参数1，/etc/passwd作为参数2，这样就创建了一个root权限用户，就能实现提权了

4.构造参数

┌──(root💀kali)-[~]
└─# openssl passwd -1 -salt admin 123456
$1$admin$LClYcRe.ee8dQwgrFc5nz.

-1 的意思是使用md5加密算法
-salt 自动插入一个随机数作为文件内容加密
admin 123456 用户名和密码

根据/etc/passwd的格式，修改一下，然后存入一个文件里，这个文件就是参数1

echo 'admin:$1$admin$LClYcRe.ee8dQwgrFc5nz.:0:0::/root:/bin/bash' >> /tmp/passwd
    # 用户名:密码:uid:gid:家目录:登陆后使用的shell

sudo ./test /tmp/passwd /etc/passwd

5.切换用户，提权成功
